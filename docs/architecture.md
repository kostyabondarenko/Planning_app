# Архитектура проекта

Здесь описывается, как устроено наше приложение.

## Основные принципы
1. Читаемость кода.
2. Актуальная документация.
3. Следование правилам из `agents.md`.

## User Flow
Пользователь заходит в проект, видит README.md и начинает работу.
Для агентов предусмотрен специальный протокол входа через чтение документации.

### Аутентификация
- Пользователи могут регистрироваться (`/auth/register`) и входить в систему (`/auth/login`).
- После входа выдается JWT-токен для доступа к защищенным ресурсам.
- В интерфейсе доступны страницы `/login` и `/register`.
