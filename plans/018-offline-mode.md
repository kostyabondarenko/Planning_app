# План: Офлайн-режим PWA (отложенный)

## Общее описание

Добавить service worker и кэширование, чтобы PWA открывалась без интернета и показывала последнее состояние. Синхронизация изменений при восстановлении сети.

**Приоритет:** низкий. Выполняется **после** всех остальных планов (014–017) и только если появится реальная потребность.

---

## Входящие требования

1. PWA открывается офлайн — показывает последний кэш целей/задач
2. Офлайн-изменения (чекбокс выполнен, новая задача) — в очереди, отправляются при восстановлении сети
3. Конфликты: «последняя запись побеждает» (простая стратегия, без CRDT)
4. Индикатор офлайн-режима в UI (баннер сверху)

---

## Промпт для запуска плана

```
Выполни план plans/018-offline-mode.md.

ВАЖНО: это последний план, зависит от всего предыдущего. Перед началом 
уточни у пользователя — действительно ли нужен офлайн-режим, или PWA 
в онлайн-режиме достаточно.

Используй скиллы: pwa-nextjs (обнови при необходимости)
```

---

## Краткий план

1. **Service worker** — через `next-pwa` или `@serwist/next` (для Next.js 16)
2. **Стратегии кэширования:**
   - Статика (`_next/static`) → CacheFirst
   - GET API (`/goals`, `/tasks`) → NetworkFirst с fallback на кэш
   - POST/PUT/DELETE → очередь в IndexedDB, Background Sync
3. **IndexedDB** для очереди мутаций (библиотека `idb`)
4. **UI:**
   - Баннер «Вы офлайн» при `!navigator.onLine`
   - Badge «Не синхронизировано» на задачах из очереди
5. **Backend:** добавить `updated_at` + optimistic locking (`If-Match` header) для разрешения конфликтов
