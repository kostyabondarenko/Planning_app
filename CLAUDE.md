# Руководство для AI-агентов

## Перед началом работы

1. Прочитай `docs/product.md` — что делает продукт
2. Прочитай `docs/tech.md` — стек и версии
3. Прочитай `docs/architecture.md` — как устроен код

## Правила работы

### Простые задачи
- Выполняй сразу, следуя best practices
- Коммить с понятным сообщением

### Сложные задачи
- Создай план в `plans/###-description.md`
- Дождись одобрения
- После выполнения обнови документацию

### Что считается сложным
- Изменения в нескольких модулях
- Новые API endpoints
- Изменения архитектуры
- Работа с безопасностью

## Обновление документации

При значимых изменениях обновляй `docs/`:
- Новые модули → `tech.md`
- Изменения структуры → `architecture.md`
- Новые фичи → `product.md`

## Формат ответа

```
1. Что сделал (результат)
2. Какие файлы изменил
3. Как проверить
```
